# Learning Strategy (Kuba Edition)

## Core Principle
Understanding is required for motivation.
No understanding → frustration → disengagement.

---

## Learning Pipeline

1. Quick explanation
2. Immediate application (build / test)
3. Partial understanding is enough to proceed
4. Deeper understanding only after it works

---

## Rule: Understand Just Enough
Do not aim for full theoretical mastery at the start.

Goal:
- Understand enough to use it
- Improve understanding iteratively

---

## Anti-Stuck Rule
If something is not understood within ~30 minutes:
- simplify the problem
- change explanation source
- ask for help

Never:
- brute-force confusion
- stack more theory on top of confusion

---

## Simplification Strategy
If confused:
- isolate smaller part
- remove complexity
- test minimal version

---

## Analog Learning Mode

Preferred method:
- build simple circuits
- measure behavior
- observe changes

Only then:
- attach theory to observation

Avoid:
- starting with heavy math

---

## Motivation Maintenance

To keep engagement:
- ensure visible progress
- create small wins
- avoid long "blind" learning phases

---

## Failure Handling

If understanding fails:
- switch approach
- reframe problem
- revisit later

Do not force continuous failure loops

---

## Summary
- Understand enough, not everything
- Apply immediately
- Simplify when stuck
- Use experimentation for learning
- Maintain progress at all costs
