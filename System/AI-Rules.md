# AI Vault Rules

These are the operating rules for what gets stored in this vault.

## 1. Save only useful context
Only save information that is likely to matter later.
Do not save random chatter, filler, or one-off nonsense unless it becomes relevant.

## 2. Prefer summaries over dumps
Store cleaned summaries, not raw transcript copies.
The goal is retrieval, not hoarding.

## 3. Separate stable vs temporary information
- Stable facts go to `Identity/`, `Projects/`, `Knowledge/`, or `People/`
- Temporary or session-bound material goes to `Logs/Session-Summaries/`
- Unsure items may go to `Inbox/` first

## 4. Update existing notes before creating duplicates
If a note for the same subject already exists, extend or revise it instead of making a fragmented clone.

## 5. Keep notes human-readable
Every note should be readable by the user without extra explanation.
No cryptic AI-only shorthand.

## 6. Prefer structure
When possible, notes should include:
- what it is
- why it matters
- current state
- next useful detail or decision

## 7. Do not over-collect personal details
Only save details that are useful for future context, planning, projects, preferences, or continuity.
Avoid unnecessary intimate or overly invasive logging.

## 8. Promote important recurring patterns
If something shows up repeatedly in session summaries, move it into a stable note.

## 9. Archive entropy
If a note becomes outdated, rewrite it, move it, or archive it.
This vault should stay useful, not become digital compost.

## 10. Reference this file when organizing new notes
This file is the source of truth for how the AI should curate the vault.
