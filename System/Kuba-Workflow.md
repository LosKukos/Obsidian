# Kuba Workflow System

This workflow is designed specifically for Kuba's working style.

---

## Core Principle
Progress = Motivation
Stagnation = Drop

The system exists to maintain continuous perceived progress.

---

## 1. Project Start Rule
- Start fast
- Do not overplan
- Get something working ASAP (even if ugly)

Goal: trigger early progress

---

## 2. "Good Enough" Rule
If it works → it is DONE (v1)

- No premature optimization
- Improvements go to v2, not now

---

## 3. Anti-Stuck Protocol

If stuck for ~1 hour:

1. Clearly define the problem
2. Try 1–2 approaches
3. If still stuck → escalate:
   - Ask AI / external help
   - Change approach completely

Never:
- grind blindly for hours
- repeat same failed attempt

---

## 4. Overengineering Brake
Before adding complexity, ask:
"Does this help me finish the current version?"

If NO → skip
If YES → proceed

---

## 5. Micro-Completion Strategy
Break work into small finishable units.

Examples:
- "Display works"
- "Button input works"
- "Data logs correctly"

Each micro-completion = reward trigger

---

## 6. Project Limit Rule
Max 3 active projects at a time.

If exceeded:
- pause or kill one project

---

## 7. Kill or Commit Rule
If a project is stuck long-term:

Choose one:
- Finish a minimal version
- Officially abandon it

Never leave projects in undefined state.

---

## 8. Flow Exploitation
To enter flow:
- Start with something easy
- Create quick wins
- Avoid starting with hard blockers

---

## 9. Reward Awareness
Completion gives:
- Ego boost
- Mental relief

Use this intentionally:
- chase completion moments
- stack them frequently

---

## 10. Assistance Usage
Using external help is allowed and encouraged.

Rule:
- Understand at least surface level
- Optimize later if needed

Goal: finish first, refine later

---

## Summary
- Start fast
- Keep progress visible
- Avoid getting stuck too long
- Finish often
- Improve later

This system prioritizes completion over perfection.
